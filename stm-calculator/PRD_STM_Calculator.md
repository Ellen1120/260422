# PRD: STM 기반 시험 자원 계산기 (Lab Resource Calculator)

**버전:** 0.1 (초안)
**작성일:** 2026-04-23
**상태:** 검토 전

---

## 1. 배경 및 문제 정의

의약품 QC 시험 준비 시, 담당자는 각 제품별 STM(Standard Test Method) 문서를 직접 읽고 필요한 시약 조제량과 초자류를 수작업으로 계산한다. 이 과정은 다음의 문제를 야기한다.

- **오류 가능성**: 배치 수 증감, 농도 변경 시 재계산 실수
- **시간 낭비**: 문서 탐색 + 계산을 매번 반복
- **지식 의존**: 경험 있는 담당자만 빠르게 처리 가능

이 앱은 STM Word 문서를 파싱하여 지식베이스를 구축하고, 입력값(제품·함량·시험항목·배치 수)을 받아 **필요한 용액 조제량 및 초자 목록을 즉시 산출**한다.

## 2. 목표

| #  | 목표                                        | 측정 기준                        |
| -- | ------------------------------------------- | -------------------------------- |
| G1 | STM 문서에서 시험 정보를 자동 추출          | 주요 시험항목 추출 정확도 ≥ 90% |

## 4. 기능 요구사항

### 4-A. STM 문서 파싱 및 지식베이스 구축

| ID   | 요구사항                                                                                                                     |
| ---- | ---------------------------------------------------------------------------------------------------------------------------- |
| F-A1 | `STM/` 폴더 내 `.docx` 파일을 로드하여 분석한다                                                                          |
| F-A2 | 각 문서에서**제품명, 함량(규격), 시험항목** 목록을 추출한다                                                            |
| F-A3 | 각 시험항목에 대해 다음을 추출한다: (1) 필요 용액 이름 및 조제법, (2) 1배치 기준 용량(mL/g 등), (3) 필요 초자류 종류 및 수량 |
| F-A4 | 추출 결과를 구조화된 JSON 지식베이스로 저장한다                                                                              |
| F-A5 | STM 폴더에 파일 추가/변경 시 지식베이스를 갱신(재파싱)할 수 있다                                                             |
| F-A6 | 파싱 실패 항목은 경고와 함께 표시하고, 나머지는 정상 제공한다                                                                |

### 4-B. 계산 엔진

| ID   | 요구사항                                                             |
| ---- | -------------------------------------------------------------------- |
| F-B1 | 입력: 제품명, 함량, 시험항목(복수 선택 가능), 배치 수                |
| F-B2 | 출력: 시험항목별 용액 조제량 (배치 수 × 1배치 기준량 + 여유분 계수) |
| F-B3 | 출력: 초자류 목록 (품목, 규격, 수량)                                 |
| F-B4 | 여유분 계수는 설정에서 조정 가능하다 (기본값: 10%)                   |
| F-B5 | 동일 시험항목을 여러 제품에 동시 적용할 때 초자 수량을 합산한다      |
| F-B6 | 계산 근거(어느 STM, 몇 페이지 기준)를 결과와 함께 표시한다           |

### 4-C. 사용자 인터페이스

| ID   | 요구사항                                                                     |
| ---- | ---------------------------------------------------------------------------- |
| F-C1 | 웹 기반 단일 페이지 앱 (로컬 실행 또는 사내 배포)                            |
| F-C2 | 제품 선택 → 함량 선택 → 시험항목 체크박스 → 배치 수 입력 순서의 단계형 폼 |
| F-C3 | 결과 화면: 용액 조제 목록 테이블 + 초자 목록 테이블                          |
| F-C4 | "인쇄" 버튼으로 결과를 브라우저 인쇄 또는 PDF 저장                           |
| F-C5 | 계산 이력을 세션 내에서 저장하고 재조회 가능                                 |

### 4-D. 관리 기능

| ID   | 요구사항                                                           |
| ---- | ------------------------------------------------------------------ |
| F-D1 | STM 파싱 결과를 관리 화면에서 확인하고 수동 수정할 수 있다         |
| F-D2 | 자동 추출이 불완전한 항목에 수동으로 데이터를 입력·저장할 수 있다 |
| F-D3 | 여유분 계수 등 전역 설정을 UI에서 변경할 수 있다                   |

## 5. 데이터 모델 (초안)

```
Product
  ├── name: string              (예: "아세트아미노펜정")
  ├── strengths: Strength[]
  └── Strength
        ├── label: string       (예: "500mg")
        └── testItems: TestItem[]

TestItem
  ├── name: string              (예: "함량시험", "용출시험")
  ├── solutions: Solution[]
  └── glassware: GlasswareItem[]

Solution
  ├── name: string              (예: "0.1N 염산 용액")
  ├── volumePerBatch_mL: number
  ├── preparation: string       (조제 방법 요약)
  └── sourceRef: string         (STM 파일명 + 페이지)

GlasswareItem
  ├── name: string              (예: "100mL 메스플라스크")
  ├── spec: string
  ├── countPerBatch: number
  └── sourceRef: string
```

## 6. 기술 스택 (제안)

| 계층            | 선택지                           | 비고                                 |
| --------------- | -------------------------------- | ------------------------------------ |
| 문서 파싱       | Python +`python-docx`          | `.docx` 텍스트/테이블 추출         |
| LLM 보조 추출   | Claude API (claude-sonnet-4-6)   | 비정형 텍스트에서 구조화 데이터 추출 |
| 백엔드          | Python FastAPI                   | 파싱, 계산, API 제공                 |
| 프론트엔드      | React + Vite (또는 단순 HTML/JS) | 사내 배포 용이성 우선                |
| 지식베이스 저장 | JSON 파일 또는 SQLite            | 별도 DB 서버 불필요                  |

## 7. 주요 사용자 흐름

```
[관리자]
STM 폴더에 .docx 파일 추가
        ↓
앱이 문서 파싱 → 지식베이스 저장
        ↓
파싱 결과 검토 화면에서 오류 항목 수동 보완

[QC 담당자]
제품 선택 → 함량 선택
        ↓
시험항목 체크 (예: 함량시험 ✓, 용출시험 ✓)
        ↓
배치 수 입력 (예: 3)
        ↓
결과 화면: 조제 용액 목록 + 초자 목록
        ↓
인쇄 / PDF 저장
```

## 9. 우선순위 및 단계별 계획

### Phase 1 — MVP (핵심 흐름 검증)

- STM 1~2개 문서 수동 파싱 → JSON 직접 작성으로 지식베이스 구성
- 계산 엔진 구현 (배치 수 × 단위량 + 여유분)
- 단순 웹 UI (제품/함량/항목 선택 → 결과 표시)

### Phase 2 — 자동 파싱

- `python-docx` 기반 구조화 추출 + LLM 보조로 비정형 텍스트 처리
- 파싱 검토 화면 구현

### Phase 3 — 고도화

- 복수 제품 동시 계산 / 초자 합산
- 인쇄용 레이아웃
- STM 변경 감지 및 자동 재파싱
